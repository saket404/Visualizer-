    About

    The final scope is to design an interface such that a user can visualize the network, the users and the interactions. The user should be able to tweak parameters like vulnerabilities for the hosts and connections between users. Furthermore, the user should be able to "play" the simulator and view metrics like the vulnerability for the overall system. The frontend should be designed in such a way that it is easy to plug in any simulator (API for backend in Python). Divide into 3 panels: Request input file only JSON format. When the user submit file you can store it in a folder name “upload”. The file will then be used by the network visualizer panel and the simulator panel. Second panel is the network visualizer where users click create and a network graph emerges where there are two type of nodes the “machine” and “user” nodes and their connections. User can choose to see only user nodes and its links or machine links only or both (you can choose different color). The properties should be shown as a label for each of the nodes and their links. Users should be able to add new nodes and edit old nodes properties and names (use input file and edit input file?). The third panel is the Simulator panel. When the user click on simulate with the input of time the plugged in simulator would use the input file as its input and return the compromised entities number. The panel should show a live graph of (compromised being the y axis and simulation time on the x axis). User can stop, pause and play the simulation.
